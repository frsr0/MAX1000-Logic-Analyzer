# OLS MaxScope — Logic Analyzer for MAX1000

8-channel logic analyzer with SDRAM deep capture, BRAM fast capture, rolling capture,
protocol decode (UART/I2C/Modbus), protocol trigger, and signal generator.

![GUI screenshot](docs/gui.png)

## Features

- **8 logic channels**, up to **48 MHz** sample rate
- **Deep capture**: up to 500,000 samples via SDRAM
- **Fast capture**: 1,024 samples via BRAM (48 MHz, no SDRAM needed)
- **Rolling capture**: continuous acquisition, PC-buffered, configurable buffer size
- **Protocol trigger**: arm on UART byte match at configurable baud rate
- **Edge trigger**: rising/falling edge on any channel
- **Generator**: UART / I2C / Modbus output on any GPIO pin
- **Protocol decode**: UART, I2C, Modbus with annotation on waveform
- **Data logger**: CSV logging with trigger events
- **Raw mode**: 4× faster readout (1 byte/sample instead of 4)
- **Export**: OLS format, Sigrok SR format, clipboard
- **Live waveform**: point-by-point drawing during capture with scrolling
- **Measurement markers**: click to place M1/M2, delta time shown

## Hardware Requirements

- **MAX1000** board (10M08SAU169C8G, 8 MB SDRAM)
- USB cable (for JTAG programming + UART serial)
- Optional: signals to probe on GPIO[0..7]

### Pin Assignments

| Signal | MAX1000 Pin | Description |
|--------|-------------|-------------|
| CLK | H6 | 12 MHz system clock |
| UART_RX | A4 | USB-UART RX (FTDI) |
| UART_TX | B4 | USB-UART TX (FTDI) |
| GPIO[0..7] | M3, L3, M2, M1, N3, N2, K2, K1 | Logic analyzer channels |
| sdram_* | See `vhdplus/pin_assignments.csv` | SDRAM interface |
| LED[0..7] | A8–D8 | Status LEDs |
| SEN_SDI/SEN_SPC | J7, J6 | Accelerometer I2C (LIS3DH) |

Full pin assignments in `vhdplus/pin_assignments.csv` (importable in VHDPlus IDE).

## Quick Start

### 1. Build the FPGA bitstream

**Option A: VHDPlus IDE (recommended)**

1. Download and install [VHDPlus IDE](https://vhdplus.com)
2. Open `vhdplus/OLS_Logic_Analyzer.vhdpproj`
3. **Tools → Pin Planner → Import** → select `vhdplus/pin_assignments.csv`
4. Verify pin assignments match your board
5. Click **Compile** (green arrow/play button)
6. **Program** the MAX1000 via USB (Tools → Programmer)

**Option B: Quartus Prime Lite 18.1**

1. Open `OLS_Logic_Analyzer_SDRAM.qpf` (from the original source, included for reference)
2. **Processing → Start Compilation**
3. **Tools → Programmer** → select the `.pof` from `output_files/` → Start

> **Note on SDRAM_PLL.vhd**: This file is generated by the Quartus ALTPLL MegaWizard. 
> If you need to change PLL settings (e.g., different clock frequencies), 
> open it in Quartus: **Tools → IP Catalog → ALTPLL** → regenerate with your parameters.
> The current configuration: 12 MHz input → c0=48 MHz, c1=120 MHz.

### 2. Install the Python host app

Requirements: **Python 3.10+**

```bash
pip install pyserial
```

Or using the provided requirements file:

```bash
pip install -r requirements.txt
```

### 3. Run

```bash
python host/OLS_Console.py
```

The app will:
1. Scan for the MAX1000 on all COM ports
2. Auto-connect when detected
3. Show the main GUI window

If auto-connect fails, click **Scan** to list available ports, select the correct COM port, and click **Connect**.

## Usage

### Capture modes

| Mode | Description |
|------|-------------|
| **Normal** | Single capture, configurable samples and rate. Supports edge trigger or protocol trigger. |
| **Rolling** | Continuous capture into a PC-side circular buffer. Data scrolls through the waveform view. Click **Stop** to freeze. |
| **Fast** | BRAM-only capture (no SDRAM). Up to 1,024 samples at 48 MHz. |

### Trigger modes

| Mode | Description |
|------|-------------|
| **Off** | Capture starts immediately when armed |
| **Rising** | Capture triggers on 0→1 edge on selected channels |
| **Falling** | Capture triggers on 1→0 edge on selected channels |
| **Protocol** | Capture triggers when a UART byte matches the configured value on the selected channel |

### Generator

| Protocol | Description |
|----------|-------------|
| **UART** | Send ASCII text at configurable baud rate on any GPIO pin |
| **I2C** | Send I2C frames (address + data) with configurable speed |
| **Modbus** | Send Modbus RTU frames with slave address, function code, and data. CRC-16 appended automatically. |

### Data Logger

1. Configure rate, samples, and trigger mode
2. Set CSV file path
3. Click **Arm** — each trigger event captures data and appends a row to the CSV
4. The trigger count shows live feedback
5. Click **Stop** to finish

### Keyboard shortcuts

- Mouse wheel: zoom in/out on waveform
- Click waveform: set markers M1, M2 (measure time between markers)
- Fit button: zoom to show the full capture

## Protocol trigger details

The protocol trigger detects UART bytes in real time at the capture clock rate.
Configure the match byte, UART channel, and baud rate in the **Trigger** tab.
When the byte is detected, the capture starts immediately with pre-trigger data
preserved in the BRAM circular buffer.

## Raw mode

Raw mode skips sending zero bytes between samples (Channel_Groups = "1110"),
reducing readout time by 4×. Enable it automatically in rolling capture mode.
In normal capture, call `dev.raw_mode(True)` before `dev.capture()`.
